## Development Workflow

### 1. Research & Reuse (mandatory before coding)
- GitHub code search → library docs → web search (in that order)
- Check package registries (npm, PyPI) before writing utility code
- Adopt existing implementations when they solve 80%+ of the problem

### 2. Plan First
- Use planner agent for implementation plans
- Plan Mode (read-only) for research → plan doc → Normal Mode for implementation

### 3. TDD Approach
- RED (test fails) → GREEN (minimal implementation) → IMPROVE (refactor)
- Coverage 80%+ (Unit + Integration + E2E)

### 4. Code Review
- After writing code → run code-reviewer agent automatically
- CRITICAL/HIGH: must fix / MEDIUM: fix when possible

### 5. Four-File Update Rule (CRITICAL for skill changes)
When adding phases or files to the skill, always update simultaneously:
1. `SKILL.md` — frontmatter triggers, help, progress detection, new phase section, Reference Files table
2. `README.md` — overview, document list, phase descriptions, file structure, skill integrations
3. `scripts/scaffold-requirements.sh` — template copies, flag support
4. `tests/skill-structure.spec.ts` — file existence, content, cross-reference tests

Run `npx playwright test` to verify all 200 tests pass before committing.

## Code Quality

### Immutability (CRITICAL)
- Never modify existing objects. Always return new copies
- Prevents side effects, eases debugging, enables safe concurrency

### File Organization
- 200-400 lines typical, 800 max. Split by feature/domain
- Functions under 50 lines, nesting under 4 levels

### Error Handling
- Handle explicitly at every level. Never swallow errors silently
- UI-facing: user-friendly messages / Server-side: detailed context logging

### Input Validation
- Validate at system boundaries (user input, external APIs, file content)
- Schema-based validation (Zod etc.) preferred

## Security (pre-commit checklist)
- [ ] No hardcoded secrets (use env vars or secret manager)
- [ ] SQL injection prevention (parameterized queries)
- [ ] XSS prevention (HTML sanitization)
- [ ] Auth/authz verified
- [ ] Rate limiting configured
- [ ] Error messages leak no sensitive data
