# UI Design Question Bank / UIデザイン質問カタログ

UIデザインフェーズ（Phase 5）で使用する質問集。1ラウンドにつき3〜5問をバッチ出題する。
ユーザーの回答状況やプロジェクト特性に応じて適切な質問を選択する。

---

## Phase 5A: UIデザインブリーフ / UI Design Brief

### Round 5A-1: プラットフォームとスタイル / Platform & Style

1. **ターゲットプラットフォームは何ですか？（Web / モバイルアプリ / 両方）**
   What is the target platform? (Web / Mobile app / Both)

2. **Webの場合、デスクトップファーストですか？モバイルファーストですか？**
   For web: desktop-first or mobile-first?

3. **デザインスタイルの方向性は？（ミニマル / エディトリアル / ダッシュボード / プレイフル / コーポレート）**
   Design style direction? (Minimal / Editorial / Dashboard / Playful / Corporate)

4. **参考にしたいWebサイトやアプリはありますか？（URL歓迎）**
   Any reference websites or apps you admire? (URLs welcome)

5. **ブレイクポイント戦略はありますか？（例: 320px, 768px, 1024px, 1440px）**
   Do you have breakpoint requirements? (e.g., 320px, 768px, 1024px, 1440px)

### Round 5A-2: ブランドとアクセシビリティ / Brand & Accessibility

6. **ブランドのプライマリカラーは決まっていますか？（HEX値歓迎、未定でもOK）**
   Do you have a primary brand color? (HEX value welcome, undecided is OK)

7. **セカンダリカラーやアクセントカラーの希望は？**
   Any secondary or accent color preferences?

8. **タイポグラフィの好みは？（サンセリフ / セリフ / ディスプレイ / 特定フォント名）**
   Typography preference? (Sans-serif / Serif / Display / Specific font name)

9. **日本語フォントの指定はありますか？（Noto Sans JP、游ゴシック等）**
   Any Japanese font requirements? (Noto Sans JP, Yu Gothic, etc.)

10. **アクセシビリティの目標レベルは？（WCAG 2.1 AA（標準） / AAA（高レベル））**
    Accessibility target? (WCAG 2.1 AA (standard) / AAA (high level))

### Round 5A-3: 既存アセットと制約 / Existing Assets & Constraints

11. **既存のデザインシステムやFigmaライブラリはありますか？（FigmaファイルのURL歓迎）**
    Do you have an existing design system or Figma library? (Figma file URL welcome)

12. **ロゴやブランドガイドラインはありますか？**
    Do you have a logo or brand guidelines?

13. **使用するアイコンセットの指定はありますか？（Lucide / Material / Heroicons 等）**
    Any icon set preferences? (Lucide / Material / Heroicons, etc.)

14. **ダークモード対応は必要ですか？**
    Is dark mode support needed?

15. **デザインの納品形式の希望は？（Figmaファイルのみ / コード生成含む / プロトタイプ含む）**
    Preferred deliverable format? (Figma file only / Including code / Including prototype)

## ドメイン固有の補足質問 / Domain-Specific Supplements

### SaaS / ダッシュボード

16. **主要なデータビジュアライゼーションの種類は？（グラフ、テーブル、KPIカード等）**
    Key data visualization types? (Charts, tables, KPI cards, etc.)

17. **サイドバーナビゲーションとトップナビゲーション、どちらが好ましいですか？**
    Sidebar navigation or top navigation preference?

### ECサイト / マーケットプレイス

18. **商品画像の品質・サイズ要件はありますか？**
    Product image quality/size requirements?

19. **カート・チェックアウトフローの参考にしているサービスは？**
    Any reference services for cart/checkout flow?

### モバイルアプリ

20. **ネイティブUIコンポーネントに準拠しますか？（iOS Human Interface / Material Design）**
    Follow native UI conventions? (iOS Human Interface / Material Design)

21. **タブバーナビゲーションの項目数は？（推奨: 3-5項目）**
    Number of tab bar items? (Recommended: 3-5 items)

### コーポレートサイト / LP

22. **ファーストビューで最も伝えたいメッセージは？**
    What is the key message for the first view / hero section?

23. **CTAボタンの主要アクションは何ですか？（問い合わせ / 申し込み / ダウンロード等）**
    Primary CTA action? (Contact / Sign up / Download, etc.)
