# MVP Light Mode Example

## Overview

This example demonstrates the output of `requirements_designer` in **Light Mode** for an MVP landing page project.

## Project: QuickForm - Simple Form Builder

### Project Charter

| Item            | Value                                                 |
| --------------- | ----------------------------------------------------- |
| Project Name    | QuickForm                                             |
| Vision          | The fastest way to create and share web forms         |
| Target Users    | Solo entrepreneurs and small business owners          |
| Success Metrics | Form creation < 2 minutes, 100 signups in first month |
| Timeline        | MVP in 3 weeks                                        |

### Scope

- Form creation with drag-and-drop fields
- Form sharing via public URL
- Response collection and CSV export

### Non-Goals (deferred)

- Payment integration
- Custom branding
- Advanced analytics

## Generated Files

| File                         | Description                                                             |
| ---------------------------- | ----------------------------------------------------------------------- |
| `functional_requirements.md` | 3 functional requirements (FR-001 to FR-003), 5 essential fields per FR |

## Mode Settings

- **Mode**: Light (MVP/PoC)
- **Phase 3 (NFR)**: Skipped
- **Phase 4C (UL)**: Skipped
- **Phase 5 (UI)**: Skipped
- **Quality Score**: 48/60 (pass line: 42/60)
- **Quality Dimensions**: Completeness, Specificity, Testability
