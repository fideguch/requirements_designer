# SaaS Full Mode Example

## Overview

This example demonstrates the output of `requirements_designer` in **Full Mode** for a SaaS project management tool.

## Project: TaskFlow - Project Management SaaS

### Project Charter

| Item            | Value                                                          |
| --------------- | -------------------------------------------------------------- |
| Project Name    | TaskFlow                                                       |
| Vision          | Enable small teams to manage projects with zero learning curve |
| Target Users    | 5-20 person startup teams                                      |
| Success Metrics | DAU/MAU > 40%, Task completion rate > 80%                      |
| Timeline        | MVP in 8 weeks                                                 |

### Scope

- Task creation and assignment
- Kanban board with drag-and-drop
- Sprint planning and tracking
- Team member management
- Real-time notifications

### Non-Goals

- Enterprise SSO integration (deferred to v2)
- Custom workflow builder (deferred to v2)
- Mobile native app (web-responsive only for v1)

## Generated Files

| File                         | Description                                  |
| ---------------------------- | -------------------------------------------- |
| `functional_requirements.md` | 5 functional requirements (FR-001 to FR-005) |
| `user_stories.md`            | 5 user stories (US-001 to US-005)            |

## Mode Settings

- **Mode**: Full
- **Phase 3 (NFR)**: Enabled
- **Phase 4C (UL)**: Enabled
- **Phase 5 (UI)**: Skipped
