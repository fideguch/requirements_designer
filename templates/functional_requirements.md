## このドキュメントについて
機能要件とは、**「システムが何をするか」**を定義したものです。各要件には優先度がついています:
Must（必須）/ Should（推奨）/ Could（あれば良い）/ Won't（対象外）

---

## Requirement ID Convention

- **FR-001** 〜 **FR-999**
- カテゴリごとに連番を振る
- 優先度: **Must** (必須) / **Should** (推奨) / **Could** (あれば良い) / **Won't** (今回は対象外)

<details><summary>書き方のヒント</summary>

**良い要件の書き方:**

✅ 「ユーザーはメールアドレスとパスワードでログインできる」（具体的）
❌ 「ユーザーはログインできる」（曖昧）

✅ 「検索結果は1秒以内に表示される（100件以下の場合）」（テスト可能）
❌ 「検索は速く表示される」（テスト不能）

✅ 「カート内商品が0件の場合、購入ボタンは非活性になる」（例外フロー明記）
❌ 「購入ボタンがある」（例外考慮なし）

**優先度の目安:**

| 優先度 | 意味 |
| ------ | ---- |
| Must   | これがないとリリースできない |
| Should | あるべきだが、なくてもMVPは成立する |
| Could  | 余裕があれば入れたい |
| Won't  | 今回は見送り（将来検討） |

</details>
